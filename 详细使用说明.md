## 1. 项目概述

跨境电商选品 AI Agent 是一个基于 Amazon 类目 Top 100 数据的智能选品分析工具。

**核心流程：**

```
上传表格 → 字段映射 → 数据清洗 → 指标计算 → 规则评分 → AI 报告 → 下载
```

**适合场景：**

- 跨境电商 Amazon 选品分析
- 类目市场调研与竞争格局判断
- 产品开发机会评估
- AI PM 面试案例分析

## 2. 环境要求与安装

### 2.1 要求

- Python 3.11+（uv 会自动管理，无需手动安装 Python）
- [uv](https://docs.astral.sh/uv/) 包管理器

### 2.2 安装 uv

**Windows**（PowerShell）：
```powershell
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
```

**Mac**（终端）：
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

安装后重启终端，验证：`uv --version`

### 2.3 跨平台运行原理

`uv` 是 Rust 编写的 Python 包管理器，在 Windows 和 Mac 上行为一致：

| 环节 | Windows | Mac | 说明 |
|------|---------|-----|------|
| Python 管理 | `%APPDATA%\.uv\` | `~/.local/share/uv/` | uv 按需自动下载 Python |
| 虚拟环境 | `.venv\` | `.venv/` | 项目隔离，不影响系统 Python |
| 依赖安装 | `uv sync` | `uv sync` | 读取 `pyproject.toml` + `uv.lock` |
| 运行脚本 | `uv run xxx` | `uv run xxx` | 自动激活 .venv 后执行 |
| 一键启动 | 双击 `run.bat` | `./run.sh` | 脚本内调用 uv sync + streamlit |

**打开顺序：**

```
第 1 步：安装 uv（一次性，约 30 秒）
  Windows: PowerShell 运行 irm ... | iex
  Mac:     终端运行 curl ... | sh

第 2 步：打开终端，进入项目目录
  cd aiagent

第 3 步：安装依赖（一次性，约 1 分钟）
  uv sync

第 4 步（可选）：配置 AI
  cp .env.example .env  →  编辑填入 DeepSeek API Key

第 5 步：启动
  Mac:   ./run.sh（或双击 run.sh）
  Win:   双击 run.bat

第 6 步：浏览器自动打开（或手动访问 http://localhost:8501）
```

**两种启动方式对比：**

| 方式 | 适合 | 原理 |
|------|------|------|
| 一键脚本 `run.sh` / `run.bat` | 日常使用 | 自动 uv sync + 启动 Streamlit |
| 手动 `uv run streamlit run app.py` | 开发者 | 跳过 uv sync，直接启动 |

### 2.4 安装项目依赖

```bash
cd aiagent
uv sync
```

### 2.5 配置 AI（可选）

```bash
cp .env.example .env
```

编辑 `.env`，填入 DeepSeek API Key：

```
DEEPSEEK_API_KEY=sk-你的key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
```

### 2.4 启动

```bash
uv run streamlit run app.py
```

## 3. 数据准备

### 3.1 数据来源

使用**卖家精灵**导出 Amazon 类目 Top 100 产品表格。

### 3.2 必要字段

| 字段 | 中文别名 | 说明 | 必须 |
|------|---------|------|------|
| `title` | 商品标题、产品标题、名称 | 产品名称 | ✅ |
| `price` | 价格、售价 | 售价（美元） | ✅ |
| `rating` | 评分、星级 | 1-5 星评分 | ✅ |
| `review_count` | 评论数、评论数量、评价数 | 评论数量 | ✅ |
| `brand` | 品牌 | 品牌名称 | 推荐 |
| `rank` | 排名、类目排名 | 类目排名 | 推荐 |
| `asin` | ASIN、商品编码 | Amazon ASIN | 推荐 |

## 4. 功能详解

### 4.1 侧边栏

| 配置 | 说明 |
|------|------|
| 目标市场 | US / UK / DE / JP / CA |
| 类目名称 | 自由填写 |
| 目标售价区间 | 参考标注（当前版本不过滤数据） |
| 文件上传 | CSV 或 XLSX |
| 示例数据 | 一键加载 |
| 历史记录 | 最近 10 次分析，点击查看 |

### 4.2 基础指标卡片

产品数 / 平均价格 / 中位价格 / 平均评分 / 评论总数

### 4.3 价格带分布

柱状图：0-9.99 / 10-19.99 / 20-29.99 / 30-49.99 / 50-99.99 / 100+

### 4.4 评论数分布

柱状图：0-99 / 100-499 / 500-999 / 1000-4999 / 5000-9999 / 10000+

### 4.5 品牌集中度

Top 5 品牌及占比，Top 1/3/5 总占比。

### 4.6 价格机会

- 主流价格带：产品最集中的价格区间
- 建议切入区间：中位价格 ± 20%
- 低/中/高端竞争程度

### 4.7 风险检测

自动扫描产品标题，匹配 40+ 中英文风险关键词：

| 类别 | 示例 | 风险 |
|------|------|------|
| 儿童安全 | baby, kids, 儿童 | CPC/ASTM 认证 |
| 知识产权 | disney, marvel, 专利 | 侵权 |
| 电子认证 | battery, electric, 带电 | FCC/UL/CE |
| 物流风险 | glass, liquid, 易碎 | 包装成本 |

### 4.8 综合评分

7 维度 100 分制，详见[评分模型详解](#5-评分模型详解)。

### 4.9 产品机会候选

筛选「评论数 > 中位数 + 评分 < 4.3」的产品，Top 10 作为改良候选。

### 4.10 选品报告

12 章节 Markdown 报告：基础信息 → 类目概况 → 市场需求 → 竞争强度 → 价格带 → 品牌集中度 → 产品机会 → 差异化方向 → 风险 → 评分 → 开发建议 → 验证清单。

### 4.11 下载

- Markdown 报告（.md）
- 清洗后 CSV（.csv）

文件名格式：`report_US_Pet_Supplies_2026-06-22.md`

## 5. 评分模型详解

### 5.1 维度与权重

| 维度 | 权重 | 评分逻辑 |
|------|------|---------|
| 市场需求 | 20 | 评论总量：>10 万满分，>5 万 80%，<5000 仅 20% |
| 改良机会 | 20 | 低评分(<4.3)高评论(>中位数)产品占比：>30% 满分 |
| 竞争强度 | 15 | 品牌分散、评分不成熟 → 高分 |
| 价格机会 | 15 | 价格带越分散，切入空间越大 |
| 品牌集中度 | 10 | Top 品牌占比 <10% 最高分 |
| 风险等级 | 10 | high=10%, medium=50%, low=100% |
| 数据质量 | 10 | 必要字段完整度比例 |

权重通过 `config/scoring_rules.yaml` 可配，总和必须为 100。

### 5.2 等级

| 分数 | 等级 | 建议 |
|------|------|------|
| 80-100 | 优先开发 | 立即进入产品定义 |
| 65-79 | 二次验证 | 验证供应链和差异化 |
| 50-64 | 谨慎观察 | 需更深入调研 |
| 0-49 | 不建议开发 | 风险过高或机会有限 |

### 5.3 局限性

- 仅基于 Top 100 表格，不看差评内容
- 不做利润测算（第二版）
- 不看供应链成本
- 所有结论需人工验证

## 10. 常见问题

**Q: 上传后提示字段不识别？**

编辑 `config/field_aliases.yaml` 添加卖家精灵导出的字段名。

**Q: 评分为什么偏低？**

检查：品牌集中度 / 评分过于成熟(>4.5) / 风险词命中 / 字段不完整。

**Q: AI 报告失败？**

1. 检查 `.env` 中 API Key
2. 检查网络和账户余额
3. 系统会自动回退到本地报告

**Q: 能直接用评分做决策吗？**

不能。评分只基于表格统计特征，不包含差评分析、利润测算、供应链验证。所有结论需人工确认。

**Q: 从不同目录启动后历史记录不见了？**

已修复，DB_PATH 自动使用绝对路径，不受启动目录影响。

## 11. 架构说明

### 11.1 技术选型

| 层 | 技术 | 理由 |
|---|------|------|
| 包管理 | uv | 全局规则 |
| 前端 | Streamlit | 零前端依赖 |
| 数据处理 | Pandas | CSV/XLSX 标准库 |
| AI | DeepSeek API | OpenAI 兼容 |
| 存储 | SQLite | 零配置 |
| 配置 | YAML + .env | 可读 + 安全 |

### 11.2 为什么第一版不做

| 不做 | 原因 |
|------|------|
| LangGraph | 流程简单，纯函数足够；面试时作为第二版规划讨论 |
| FastAPI | Streamlit 内置后端 |
| Playwright | 聚焦表格分析，爬虫留到第二版 |
| 多 Agent | 单 Agent 已满足，多 Agent 增加不稳定性 |

### 11.3 数据流

```
CSV/XLSX → load_table → map_fields → clean_product_data
  → metrics (basic + brand + price_opp)
  → risk_checker
  → scoring (7 dims × weights)
  → ai_reporter / report_generator
  → db.save → Streamlit 展示 + 下载
```

### 11.4 核心原则

- **数值计算 = 代码**：Pandas 算所有数值
- **语义分析 = AI**：DeepSeek 解释数据、生成建议
- **AI 收全量数据**：100 行不给截断
- **容错第一**：缺字段不崩，无 Key 不退场
- **透明可追溯**：每个评分都有 reason
